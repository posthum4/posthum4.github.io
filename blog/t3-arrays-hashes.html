Arrays and Hashes

5/17/2015

As a beginner learning Ruby it is not clear at first glance what are the differences between arrays and hashes. Even when researching the topic it took a while to find an explination that resonated with me. The best I found was to use hashes when the location of a value in the list is important, and use arrays when it is not.

Why is it difficult to tell which option to use? Becuase arrays and hashes are very similar. Both are collections that can have values added, removed, accessed, and manipulated. But there is one main difference. That difference is how the location of each value in the collection is specified. An array uses integers, starting at position 0, to assing a position to each value. Hashes use a key that can be assigned the user. This can can be any other object.

Array
Position 0 1 2 3
Value 10 20 30 40


Hashes
Key "Name" "Grade" "School" "Year"
Value Ryan 12 MSHS 2002

Using these sample data sets you can start to see where hashes and arrays differ. Arrays are good to use when you need an ordered colletion of values of undefined length, but whose position in the list is not important. Hashes are better suited for recalling a specific piece of information for the list when you already now the key value.

Let's look at an example where you want to find out what grade a particular student is in. If you were using arrays you would have to iterate over the entire list until you found that particular student's name. With hashes you could easily pull that students information by calling:

student_record["grade"] = 12
